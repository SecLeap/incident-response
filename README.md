# 网络安全应急响应知识库

本仓库以**应急响应工作闭环**为主线，而非以溯源取证为主线：让团队能够准备好、接住告警、快速定级和止血、恢复业务、协调沟通，并通过复盘持续改进。

当前仅包含目录与待办清单，后续逐步补充 Runbook、检查清单、报告模板、演练剧本和脱敏案例。

## 工作原则

- 先控制影响、保护业务连续性，再深入调查和追溯。
- 每项处置必须记录：谁在何时基于什么证据执行了什么动作，结果如何。
- 取证、溯源、恶意代码分析是支撑调查和复盘的能力，不替代响应决策。
- 仅在合法授权和隔离可控的条件下验证技术内容；不保存敏感数据与真实凭据。

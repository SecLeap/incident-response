# 应急响应知识库 TODOLIST

## P0：让响应工作可运行

- [ ] 明确事件生命周期、角色职责、值班、升级路径和战情协同机制。
- [ ] 建立事件受理、定级、工单、处置记录、关单和复盘的统一流程。
- [ ] 盘点关键业务、资产负责人、账号权限、日志源、备份和紧急联系人。
- [ ] 为勒索、账号失陷、钓鱼、Web 入侵、数据泄露编写首版 Runbook。
- [ ] 建立隔离、封禁、账号禁用、密钥轮换和回滚的变更控制规范。

## P1：让响应更快、更准

- [ ] 建立告警分流、去重、优先级、误报处置和升级标准。
- [ ] 完善终端、身份、网络、邮件、云和业务日志的最小可用基线。
- [ ] 建立影响评估、横向扩线、业务恢复验证和风险接受机制。
- [ ] 建立内部战报、管理层简报、客户/监管通报和结案报告模板。

## P2：让能力持续提升

- [ ] 建立桌面推演与技术演练计划，并跟踪改进项闭环。
- [ ] 建立响应指标：MTTA、MTTD、MTTC、MTTR、重复事件率和预案覆盖率。
- [ ] 引入自动化处置候选项，并验证收益、权限与误处置风险。
- [ ] 沉淀脱敏案例、调查方法、检测缺口和威胁狩猎假设。

## 完成定义

- [ ] 可由值班人员在压力下执行，不依赖隐性经验。
- [ ] 明确业务影响、授权边界、风险与回滚方案。
- [ ] 有证据、操作记录、验证结果和责任人。
- [ ] 经演练、真实事件或桌面推演验证并定期复审。
